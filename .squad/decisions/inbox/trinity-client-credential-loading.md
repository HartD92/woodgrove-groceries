### 2026-07-22: Use Microsoft.Identity.Web ClientCredentials array for storefront downstream token acquisition
**By:** Trinity
**What:** Configure the storefront's default AzureAd confidential-client credential with `AzureAd:ClientCredentials[0]` (`SourceType=ClientSecret`, `ClientSecret=<existing Key Vault reference>`) instead of only the legacy flat `AzureAd:ClientSecret`, and update the checked-in default AzureAd sample to use the same ClientCredentials shape.
**Why:** Sign-in succeeded through hybrid id_token issuance, but downstream token acquisition failed with IDW10109 because Microsoft.Identity.Web v3's token-acquisition credential loader uses the `ClientCredentials` collection. The deployed secret resolved successfully from Key Vault, but it was not presented in the collection used by the credential loader; the only configured credential shape in appsettings was an unloadable placeholder certificate. The explicit ClientCredentials array makes the existing secret available to the OBO/token-acquisition path without hardcoding any secret value.
